Thesis

Healthcare SaaS is just a database with a UI bolted on.

PointClickCare. Workday. Microsoft 365. Each is a database + an API + a UI. Operators pay millions per year — for the UI. Agents don't need it.

Skip the application layer

Agents call the API directly. The UI shrinks to a decision queue, not an app to learn.

Compress 12 tabs into one decision

Self-contained briefings: every dollar, room number, citation, recommendation in one card. <10s human action.

Agents under HITL governance

Read-only by default. Mutations gated by role + confidence + immutable audit chain. Kill switch global.

04 / 09
Methodology

First-principles design rules — non-negotiable.

Self-contained decisions

Every DecisionCard ships with all the data: resident name, room, dollar amount, source system, evidence, agent recommendation, confidence, three-step "if-approved" plan. The human never opens another app.

Definitive recommendations

Not "consider reviewing" — "Approve this. Here is exactly what will happen, in this order, by these times." The agent commits.

Quantified impact, every time

Dollars saved, days reduced, risk probability, regulatory citation. If it can't be quantified, the decision doesn't ship.

Confidence is conviction

≥95% → auto-approve. 80–94% → human single-tap. <80% → escalation with two-sided agent debate. Confidence is a routing dimension, not a label.

05 / 09
Architecture

One universal agent loop. Six governance levels.

Every agent — clinical, financial, M&A — runs the same loop. Behavior differs only by tools, prompts, and the governance level the operator assigns it.

1
Observe. MCP connectors stream events from PCC, Workday, M365, CMS/OIG/SAM into a typed graph.
2
Reason. Anthropic Managed Agents session — pinned agent version, SSE event stream, after_id cursor.
3
Propose. Decision draft with evidence table, citations, confidence score, and a 3-step rollback plan.
4
Gate. Governance level + confidence threshold decides: auto-approve, HITL, or escalate.
5
Act. Tool call via vault-and-proxy — agent never touches raw credentials.
6
Audit. Hash-chained, monotonically-ordered append-only log. Replayable from any point.

08 / 09
Security · Built to the Anthropic bar

Vault. JWKS. Hash chain. PHI never leaves the tenant VPC.

Vault-and-proxy credentials

Agents never see raw credentials. MCP proxy injects per request from AWS Secrets Manager. 90-day automated rotation, dual-key cutover, <15-min emergency revocation.

JWKS · RS256 · Azure Entra ID

6h key cache, auto-rotate on verify failure. JWT_SECRET fallback for service-to-service only — alg-routed, not co-mingled. RBAC on every state-changing endpoint.

Immutable audit chain

Append-only log. Monotonic sequence numbers under concurrent writes. SHA-256 hash chain — tamper-evident. Replay-safe from any point. Ships to compliance unchanged.

PHI tokenization at the boundary

Session-scoped UUID prefix on every de-id token — no cross-session collision. AWS Bedrock target deployment for in-VPC PHI processing under BAA / SOC 2 / HITRUST.
