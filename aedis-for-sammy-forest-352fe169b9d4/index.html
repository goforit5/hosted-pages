The Status Quo

Two hats, two back offices, same broken pattern

On both sides of your business, the work today is “PDF in — person re-keys — spreadsheet ages out.” This is the industry as it actually runs.

Operator hat — the facilities you run

  • AP invoice lands in a shared inbox — sits hours
  • OCR vendor reads ~70% of fields, clerk re-keys the rest
  • Operator chased over email for the GL code
  • Controller approves on Friday batch — cuts check Tuesday
  • BEC wire requests, duplicate invoices, ghost vendors caught after the fact (if at all)
  • ~3 days per invoice, 4 people, no reasoning logged

Landlord hat — the operators in your portfolio

  • Tenant operators email quarterly PDF coverage packets
  • Asset manager re-keys EBITDARM into a master spreadsheet
  • State survey results, F-tags, star ratings tracked manually (or not at all)
  • Operator distress shows up in the rent check, not the dashboard
  • No early warning on occupancy drift, quality-mix slide, or regulatory action
  • ~45-90 days to know a tenant is in trouble, 0 cross-portfolio comparability

The OCR tool reads the invoice. The PDF reader reads the coverage packet. Everything after that is people.

The Same Work, Agentified

Same data. Same controls. Agent reads it the second it lands.

Pick either hat. The pattern is identical: an agent owns the inbox, reads the message, decides what it is, and either acts and logs it or hands you a sub-10-second decision with the evidence attached.

Today — both hats

  • Invoice / coverage packet lands in shared inbox
  • OCR or PDF reader gets ~70% of fields
  • Human re-keys the rest into ERP or master sheet
  • Exceptions live in a spreadsheet nobody owns
  • Approval batched weekly — decisions logged as initials
  • Fraud / distress signals caught after the fact, if at all
  • 3 days per invoice • 45-90 days per coverage packet

Agent path — Aedis

  • Agent reads the message the second it lands
  • Vendor whitelist / operator baseline + variance check
  • GL code or EBITDARM ratio applied from 12-mo history
  • Controls tighter: every decision has cited reasoning
  • High-confidence batch auto-approved, low-confidence surfaced with evidence
  • Distress signals: occupancy drift, coverage breach, F-tag spike — surfaced before the rent check is late
  • ~15 seconds per invoice • same-day on coverage signals

Controls don’t get weaker — they get tighter, because every decision now has reasoning attached to it instead of an initial in a margin.

Beyond OCR

What the agent actually knows

An OCR tool reads characters. This agent reads context — about each facility you run, and about each operator you collect from. That is the difference between routing mail and running an SNF book.

Vendor & operator whitelist

Which vendors are real for which facility. Which operators are on which leases. Which addresses, remit-to instructions, and EINs match the record.

PO & coverage history

What a normal invoice from this vendor looks like. What a normal coverage packet from this operator looks like. Tolerance bands — learned, not configured.

GL coding & ratio history

How this facility has coded this vendor for 12 months. How this operator’s EBITDARM has trended for 12 quarters. Same code, same approver, same baseline.

Behavioral baseline, per signer

What the real CFO has and has not ever signed. What the operator’s real CFO normally certifies. New wire instructions to a generic mailbox? That’s a flag.

Distress & fraud library

Current BEC patterns in SNF AP. Current operator-distress signatures — occupancy slide + Medicaid mix-up + new survey deficiency = top-three distress signature.

Statute & covenant library

I-9 NOI — 3 business days. RAC audit — statutory clock. Lease coverage covenant — quarterly test. Cited on the card with the exact clause.

Policy version in force

Your controls captured as code. The agent knows which policy version was in effect on which day at which facility — and the audit row cites it.

Governance & Control

Governed AI — from data access
to production deployment

Every agent action is auditable, reversible, and human-approved. Autonomous preparation with human accountability preserved.

Human-in-the-Loop

Agents propose, humans approve. Every ERP, PCC, or asset-mgmt write-back shows a diff preview. No autonomous writes without explicit approval.

Complete Audit Trail

Every action logged — timestamp, user, data accessed, rationale. Three-layer audit: hooks, middleware, DB triggers. SOX-ready from day one.

Role-Based Access

Agents inherit user permissions. AP clerk sees AP queue. Asset manager sees their tenants. Principal sees the portfolio. Same RBAC you already enforce.

Regulatory Compliance

  • SOX — Immutable hash-chained logs, separation of duties, approval records
  • HIPAA — BAA-covered, PHI tokenized at the boundary if/when it touches a mailbox
  • IRS / DOL / USCIS — Statutory deadlines tracked, citations on every decision card
  • State regs & survey law — Multi-state employment + survey-action rules per facility

AI Safety Controls

  • Hallucination guard — Sources cited, confidence scored
  • Read-only default — Write requires explicit approval
  • Governance levels — Full approval → relax as trust earned
  • Kill switch — Disable any agent instantly, per facility or per tenant

Reliability & Consistency

Agents run defined workflows, not open-ended generation. Same input produces same structured output. Monitored, versioned, regression-tested.

Adoption & Change Management

Data Privacy & Governance
