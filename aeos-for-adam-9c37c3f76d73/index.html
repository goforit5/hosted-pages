05 The four pillars

What makes this safe to deploy
in a regulated domain.

Pillar 1 · Memory

Brain graph + semantic search

Every prior decision, every override, every outcome, persisted as typed nodes with embedded vectors. The system remembers context across sessions, agents, and operators. This is what "30 years of behavioral intelligence" looks like when it's actually structured.

Pillar 2 · Schemas

Typed nodes, typed edges

No free-text dumping ground. Every entity has a node_type, every relationship an edge_type, both versioned and audited. Means a claim can be traced to a source, a decision can be traced to its inputs, a model output can be tied to its evidence.

Pillar 3 · Autonomy + HITL

Propose-only L2 agents, sensitivity-routed gates

Agents operate at tiered autonomy levels. A sensitivity classifier routes consequential proposals to a reviewer console. PHI / PII is tokenized at the trust boundary and never reaches the human UI — three-layer defense pattern.

Pillar 4 · Self-improving loop

Signal Miner → Critic → Eval Harness → Shadow + Canary

Every model variant runs against a frozen evaluation set before it sees real traffic. Shadow + canary rollout. Sub-5-second auto-rollback on drift. Shipped last week. This is the part most platforms skip — and the part that makes the engine trustable over time.
