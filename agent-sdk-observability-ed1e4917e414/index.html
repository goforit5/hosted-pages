How it's wired env vars → OTLP → your backend

1. Enable in your SDK process

no code instrumentation required
# Set these on the env that runs Agent SDK (laptop, container, lambda).
# The Claude Code CLI inherits them and exports OTLP automatically.

CLAUDE_CODE_ENABLE_TELEMETRY=1
CLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1     # traces

OTEL_METRICS_EXPORTER=otlp
OTEL_LOGS_EXPORTER=otlp
OTEL_TRACES_EXPORTER=otlp

OTEL_EXPORTER_OTLP_ENDPOINT=https://api.honeycomb.io:443
OTEL_EXPORTER_OTLP_HEADERS="x-honeycomb-team=YOUR_KEY"

OTEL_METRIC_EXPORT_INTERVAL=1000
OTEL_LOGS_EXPORT_INTERVAL=1000

OTEL_SERVICE_NAME=aeos-orchestrator
OTEL_RESOURCE_ATTRIBUTES="deployment.environment=production,team=aeos"

# Optional — capture prompt + tool I/O (sensitive)
OTEL_LOG_USER_PROMPTS=1
OTEL_LOG_TOOL_DETAILS=1
OTEL_LOG_TOOL_CONTENT=1

2. Point at any OTLP backend

pick one — same env vars
# Local dev — Jaeger UI on :16686
docker run -p 4317:4317 -p 16686:16686 \
  jaegertracing/all-in-one

# Honeycomb (LLM-aware traces)
OTEL_EXPORTER_OTLP_ENDPOINT=https://api.honeycomb.io:443

# Datadog (APM + alerts)
OTEL_EXPORTER_OTLP_ENDPOINT=https://...datadoghq.com

# Langfuse — first-class Agent SDK integration
#  session replay + LLM evals + cost
OTEL_EXPORTER_OTLP_ENDPOINT=https://cloud.langfuse.com

# Multi-export: traces → Jaeger, metrics → Prometheus
OTEL_TRACES_EXPORTER=otlp
OTEL_METRICS_EXPORTER=prometheus

3. Optional — inline programmatic monitoring via hooks

.claude/settings.json
{
  "hooks": {
    "PreToolUse":    [{ "command": "./hooks/log-tool-use.sh" }],
    "PostToolUse":   [{ "command": "./hooks/cost-tracker.sh" }],
    "SessionStart":  [{ "command": "./hooks/session-init.sh" }],
    "SubagentStop":  [{ "command": "./hooks/subagent-cost.sh" }],
    "Stop":          [{ "command": "./hooks/session-summary.sh" }]
  }
}
